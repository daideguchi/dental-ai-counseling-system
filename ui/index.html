      データベース保存仕様

      SOAP記録は以下の形式でデータベースに保存されます

      テーブル構造

      counseling_sessions
      id TEXT(50) PRIMARY KEY セッション識別子
      patient_id TEXT 患者ID
      doctor_id TEXT 医師ID
      soap_note JSON SOAP記録データ
      confidence_score REAL 信頼度（0.0-1.0）

      SOAP記録フォーマット

      JSON構造
      "S": 主観的情報（患者の訴え）
      "O": 客観的所見（医師の観察）
      "A": 評価・診断（医師の判断）
      "P": 計画（治療方針）
      "confidence": 信頼度スコア

      保存時の動作

      自動処理
      1 ファイル内容を解析
      2 SOAP形式に変換
      3 品質分析を実行
      4 データベースに保存
      データベース構造は初期設定のため、運用要件に応じて変更可能です。
